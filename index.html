Sass

RUBY compiler is a must, before building!!!

In Sass, variables start with a $ followed by the variable name:

$headings-color: green;

h1 { color: $headings-color; }

Sass allows nesting of CSS rules, which is a useful way of organizing a style sheet:

nav {
  background-color: red;
  ul {
    list-style: none;
    li {
      display: inline-block;
    }
  }
}

In Sass, a mixin is a group of CSS declarations that can be reused throughout the style sheet:

@mixin box-shadow($x, $y, $blur, $c){ 
  -webkit-box-shadow: $x, $y, $blur, $c;
  -moz-box-shadow: $x, $y, $blur, $c;
  -ms-box-shadow: $x, $y, $blur, $c;
  box-shadow: $x, $y, $blur, $c;
}

div { @include box-shadow(0px, 0px, 4px, #fff); }

The @if directive works just like the if statement:

@mixin text-effect($val) {
  @if $val == danger {
    color: red;
  }
  @else if $val == alert {
    color: yellow;
  }
  @else if $val == success {
    color: green;
  }
  @else {
    color: black;
  }
}

div { @include text-effect(danger); }

Directive @for adds styles in a loop. The "start to end" excludes the end number, and "start through end" includes the end number. The #{$i} part is the syntax to combine a variable (i) with text to make a string:

@for $i from 1 through 12 {
  .col-#{$i} { width: 100%/12 * $i; }
}

The @each directive which loops over each item in a list or map. The $key variable is needed to reference the keys in the map. Otherwise, the compiled CSS would have color1, color2... in it.

$list: (color1: blue, color2: red, color3: green);
@each $key, $color in $list {
  .#{$color}-text {color: $color;}
}

The @while directive is an option with similar functionality to the while loop. After setting the CSS rule for width, $x is incremented by 1 to avoid an infinite loop:

$x: 1;
@while $x < 13 {
  .col-#{$x} { width: 100%/12 * $x;}
  $x: $x + 1;
}

Partials in Sass are separate files that hold segments of CSS code. Names for partials start with the underscore (_) character. For example, if all your mixins are saved in a partial named "_mixins.scss" import looks like:

// In the main.scss file

@import 'mixins'

Sass has a feature called extend that makes it easy to borrow the CSS rules from one element and build upon them in another:

.panel{
  background-color: red;
  height: 70px;
  border: 2px solid green;
}

.big-panel{
  @extend .panel;
  width: 150px;
  font-size: 2em;
}

To use built-in CSS functions and SASS variables use #{...} construction:

padding-top: calc(#{$header-height}/2 - #{$header-ul-li-font-size}/2);
Grid

Turn any HTML element into a grid container by setting its display property to grid:

display: grid;

To add columns to the grid, use the grid-template-columns property on a grid container:

grid-template-columns: auto 50px 10% 2fr 1fr;

To adjust the rows manually, use the grid-template-rows property:

grid-template-rows: auto 50px 10% 2fr 1fr;

To add a gap between the columns, use the grid-column-gap property:

grid-column-gap: 10px;

Add a gap in between the rows of a grid using grid-row-gap:

grid-row-gap: 5px;

Shorthand property for both rows and columns is a grid-gap:

grid-gap: $grid-row-gap $grid-column-gap;

The grid-column property is for use on the grid items themselves, where the first number for start-line (starting from 1) and the second number for end line:

grid-column: 2/4;

Same for rows:

grid-row: 1/4;

To align the content's position within its cell horizontally use the justify-self property on a grid item:

justify-self: (start/center/end);

To align an item vertically:

align-self: (start/center/end);

To align items within container all at once horizontally use justify-items on your grid container:

justify-items: (start/center/end);

Same for vertical alignment:

align-items: (start/center/end);

To group cells of your grid together into an area - give the area a custom name by using grid-template-areas on the container:

grid-template-areas:
  "header header header"
  "  .   content content"
  "footer footer footer";

You can place an item in your custom area by referencing the name you gave it:

.item1 { grid-area: header; }

You can create an area on the fly for an item without creating an areas template:

.item1 { grid-area: horizontal line start/vertical line start/horizontal line end/vertical line end; }

By using the repeat function you can specify the number of times you want your column or row to be repeated:

grid-template-rows: repeat(100, 50px); grid-template-columns: repeat(2, 1fr 50px) 20px;

To limit the size of items when the grid container changes size (minimal and maximal):

grid-template-columns: 100px minmax(50px, 200px);

To automatically insert as many rows or columns of your desired size as possible depending on the size of the container:

grid-template-columns: repeat(auto-fill, minmax(60px, 1fr));

To collapse empty rows or columns and stretch your items to fit the size of the container use auto-fit:

grid-template-columns: repeat(auto-fit, minmax(60px, 1fr));
Flexbox

Place the CSS property display:flex; on an element allows you to use other flex properties:

display: flex;

To align any children of element into rows or columns add the flex-direction property to the parent item:

flex-direction: (row/row-reverse/column/column-reverse);

To wrap items use flex-wrap on the container. The break point of where the wrapping happens depends on the size of the items and the size of the container:

flex-wrap: (nowrap/wrap/wrap-reverse);

To combine flex-wrap and flex-direction use flex-flow:

flex-flow: direction wrap;

To space the flex items along the the main axis use justify-content on the container:

justify-content: (flex-start/center/flex-end/space-between/space-around);

To align flex items along the cross axis within container use align-items:

align-items: (flex-start/center/flex-end/baseline/stretch);

The flex-grow property controls the size of items when the parent container expands:

flex-grow: (0/number);

When used, flex-shrink property allows an item to shrink if the flex container is too small. Items shrink when the width of the parent container is smaller than the combined widths of all the flex items within it. The higher the number, the more it will shrink compared to the other items in the container:

flex-shrink: (0/number);

The flex-basis property specifies the initial size of the item before CSS makes adjustments with flex-shrink or flex-grow:

flex-basis: (auto/content/size/%);

Shortcut to set several flex properties at once:

flex: grow shrink basis;

The order property is used to tell CSS the order of how flex items appear in the flex container:

order: (0/number);

The align-self property allows you to adjust each item's alignment individually, instead of setting them all at once:

align-self: (auto/flex-start/center/flex-end/baseline/stretch);

bind() creates a new function that, when called, has its 'this' keyword set to the provided value,
with a given sequence of arguments preceding any provided when the new function was called.
Support: ECMAScript >= 5 (thus >= IE9)

Unlike bind(), apply() and call():

  1. Accept additional parameters as well.
  2. Executes the function it was called upon right away.
  3. The call() and apply() methods do not make a copy of the function it is being called on.

class FilledComponent extends React.Component {
  constructor(props) {
    super(props);
    //State are defined here:
    this.state = {
      input: '',
      messages: [] 
    }
    //All methods must be bound explicitly to the component:
    this.handleChange = this.handleChange.bind(this);
    this.submitMessage = this.submitMessage.bind(this);
  }
  //Methods are defined here:
  handleChange(event) {
  //Dont change state directly. Use setState method:
    this.setState({
      //Access to event value:
      input: event.target.value
    });
  }
  submitMessage() {
    const currentMessage = this.state.input;
    this.setState({
      input: '',
      messages: [...this.state.messages, currentMessage]
    });
  }
  render() {
    //Write JS code here before return without brackets "{}"
    return (
      <div>
        <h2>Type in a new Message:</h2>
        <input
          value={this.state.input}
          onChange={this.handleChange}/><br/>
        <button onClick={this.submitMessage}>Submit</button>
        <ul>
          //Render elements depending on array values amount
          {this.state.messages.map( (message, idx) => { 
              return (
                //Dont forget to set key property
                <li key={idx}>{message}</li> 
              )
            })
          }
        </ul>
      </div>
    );
  }
};
